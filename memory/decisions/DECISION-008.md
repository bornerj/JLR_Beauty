Status: ACTIVE
Date: 2026-06-11
Context: PLAN-0012 — todos os textos de marketing das páginas públicas (títulos, subtítulos, CTAs, depoimentos) estavam hardcoded nos componentes React. Qualquer alteração exigia edição de código e novo deploy. Alguns textos têm formatação composta (parte em gradiente dourado, parte normal) que não pode ser representada como string simples.
Decision: (1) Armazenar todos os textos editáveis em `Setting.key = "public.pageTexts"` como JSON, seguindo o padrão já existente de `branding` e `mediaSlots`. (2) Representar textos com estilo parcial como array de segmentos `{ text: string; style: StyleId }` com vocabulário fechado de 6 estilos mapeados para classes CSS do projeto (`default`, `gold-gradient`, `primary`, `gold`, `bold`, `uppercase`). (3) Sem editor WYSIWYG externo (Tiptap ou similar) — editor proprietário simples com lista de segmentos + dropdown de estilo. (4) Textos de serviços, produtos e memberships ficam fora do escopo — editados pelos seus próprios módulos Admin.
Consequences: Zero dependência de biblioteca de editor; estilos controlados por código (novos estilos requerem deploy); sistema extensível — novos campos adicionados ao catálogo aparecem automaticamente no Admin; fallback automático para valores padrão do catálogo quando chave ausente no banco.
